◆ ser_output_flush()

void ser_output_flush ( void )

Output queue vexSerialWriteBuffer doesn't seem to be very thread safe, so the system daemon flushes an intermediary buffer once before vexTasksRun calls to write add to the queue. Flushing is optimized because we have underlying access to the write buffer, as opposed to calling queue_recv a bunch of times
